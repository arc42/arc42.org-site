With help embeds the official arc42 explanations in every section, ideal when you are new to the template. Plain gives you the bare structure. All formats are generated from their AsciiDoc sources in the GitHub repository.

Which format is right for you?

AsciiDoc
Powerful yet simple markup, used by arc42 itself. Ideally suited to architecture documentation, plays well with docToolchain.
Markdown
Widespread, simple markup. Strict variants restrict it further; multi-page variants split every chapter into its own file.
Word (.docx)
Microsoft Word, also usable with LibreOffice or OpenOffice.
GitHub Markdown
GitHub Flavored Markdown, used on GitHub.com and GitHub Enterprise.
LaTeX
The document preparation system, for those who need beauty and are willing to invest some effort.
reStructuredText
Heavily used by Read the Docs and in the Python world.
Textile
Another simple markup language.
HTML
For viewing the template only, not for working with it.

Specialised and tool formats

Community-contributed versions for modelling tools and wikis.

  • ConfluenceGenerated with help, flat or structured (one page per section), DE and EN. For current versions we recommend authoring in AsciiDoc and syncing with asciidoc2confluence.
  • Enterprise ArchitectThe complete arc42 structure in UML diagrams and text, with the official explanations as notes. EAPX and QEA files, DE and EN. Contributed by Raphael Dumhart.
  • DoxygenFor generating documentation from annotated sources (C++, C#, Java, Python and more). EN only. Contributed by Stephan Lessing.
  • IBM RhapsodyThe complete arc42 structure modelled in UML/SysML, with profile, RPE report template and a worked example. Contributed by Niranjan SK.

We would love to support more tools. If you use arc42 with one, please consider contributing.
