Layout study · Homepage & Download

Two revised directions for each page, drawn against the critique.

The Impeccable review scored the homepage 25/40 — functional, right vocabulary, but let down by one structural gap and several accessibility snags. The headline verdict: the site's primary job is to hand people the template, yet neither page opens with a download.

Below are the findings I'm designing against, then two alternative layouts per page — a pragmatic conversion-first take and a more confident editorial take. Switch between them with the revision tabs. Every mockup uses the real arc42 palette (navy #374769, steel #4b7ba3); image areas are shown as wireframe placeholders.

P1 · CTA

No download button

Primary purpose buried behind "Overview" and "read more…". High friction to the one action that matters.

P1 · A11y

Invalid HTML nesting

<a><button> breaks screen readers & keyboard nav. Must be a real link styled as a button.

P1 · Contrast

Illegible colour pairs

Ukraine banner ~1.03:1; button hover PaleGoldenrod-on-steel 3.6:1. Both fail WCAG AA.

P2 · Clarity

Five identical "read more…"

No context for screen-reader users; forces reliance on visual proximity.

P2 · Rhythm

Unbalanced grid + double H1

News column has no image/button; # Learn more creates a second H1.

MIN · Noise

Heartbeat animation

Distracting on an otherwise static, professional page; excessive bolding adds visual noise.

Revision

A · Conversion split

Intent

Put the template one click from the fold. Hero states the value; the panel next to it lets power users grab a format immediately.

Fixes

  • P1 Primary "Download the templates" CTA + a quick-pick panel in the hero
  • P1 All buttons are real links; navy button, white hover text (AA)
  • P1 Ukraine banner → navy bar, legible white text
  • P2 "Get the template / See trainings / Explore the method" replace "read more…"
  • P2 3-card balanced row; News demoted to a slim strip; one H1

Trade-off

Densest option — the hero carries a headline, sub, two CTAs and a panel. Needs disciplined spacing on mobile (panel stacks under).

Revision

B · Editorial blueprint

Intent

A confident, print-inspired page. Type does the work; the wordmark is the hero; sections become an indexed contents list like a spec sheet.

Fixes

  • P1 Single, unmissable "Get the templates" CTA in the hero
  • P1 No nested button markup anywhere; links throughout
  • P1 Ukraine message as a legible footnote, not a colour clash
  • P2 Numbered index carries real order + descriptive verbs, zero "read more…"
  • P2 One H1 (the wordmark); imagery removed → no grid imbalance; no heartbeat animation

Trade-off

Boldest departure from today's look. Drops the feature imagery, so it leans entirely on typography and copy quality to feel premium.

Revision

B · Guided two-pane picker

Intent

Turn "read a 12×12 grid" into "answer two questions." Pick language, pick format, get exactly one clear result with plain/with-help side by side.

Fixes

  • P1 Two obvious download buttons, always in view; no hunting the grid
  • P2 "Plain vs with-help" finally explained at the point of choice
  • P2 Niche tool formats (EA, Rhapsody…) demoted to a secondary shelf
  • MIN Removes the stacked .notice banners entirely

Trade-off

Needs a little JavaScript to be truly interactive, and power users who want to eyeball everything lose the all-at-once table (link one from the picker).

How these map to the fixes

Each direction is non-destructive — these are layout studies, not committed changes. The homepage lives in Jekyll frontmatter (feature_row blocks), so REV A is a moderate template edit; REV B is a larger rework of the splash into a custom layout. My recommendation: ship the homepage REV A + download REV A first (they clear every P1 with the least churn), then trial the editorial REV B on a branch if you want a bolder identity.
